# Story 2.3: Test Triangle with Dynamic Rendering

Status: ready-for-dev

## Story

As a developer,
I want a colored triangle rendering to the screen,
so that I validate the entire Vulkan pipeline end-to-end.

## Acceptance Criteria

1. Vertex + fragment shader (hardcoded triangle positions + colors in shader, no vertex buffers)
2. Graphics pipeline created with dynamic rendering (no VkRenderPass, no VkFramebuffer)
3. Frame rendering: acquire swapchain image -> begin dynamic rendering -> bind pipeline -> draw(3) -> end rendering -> present
4. Synchronization via fences + semaphores (image available, render finished)
5. Window clear color visible behind triangle
6. No validation layer errors

### Dynamic Rendering — No VkRenderPass, No VkFramebuffer

The physical device was selected requiring `dynamicRendering = VK_TRUE` (Vulkan 1.3 core). This project never creates `VkRenderPass` or `VkFramebuffer` objects.

Instead:
- Pipeline: chain `VkPipelineRenderingCreateInfo` in `pNext`, set `renderPass = VK_NULL_HANDLE`
- Drawing: use `vkCmdBeginRendering(cmd, &renderingInfo)` / `vkCmdEndRendering(cmd)`
- Layout transitions: **manual** via `vkCmdPipelineBarrier2` — dynamic rendering does NOT transition layouts for you

### Image Layout Transition Barriers (sync2)

Use `VkImageMemoryBarrier2` + `VkDependencyInfo` + `vkCmdPipelineBarrier2` (non-KHR, Vulkan 1.3 core):

**Before rendering** — `UNDEFINED -> COLOR_ATTACHMENT_OPTIMAL`:
```cpp
VkImageMemoryBarrier2 barrier{};
barrier.sType            = VK_STRUCTURE_TYPE_IMAGE_MEMORY_BARRIER_2;
barrier.srcStageMask     = VK_PIPELINE_STAGE_2_COLOR_ATTACHMENT_OUTPUT_BIT;
barrier.srcAccessMask    = VK_ACCESS_2_NONE;
barrier.dstStageMask     = VK_PIPELINE_STAGE_2_COLOR_ATTACHMENT_OUTPUT_BIT;
barrier.dstAccessMask    = VK_ACCESS_2_COLOR_ATTACHMENT_WRITE_BIT;
barrier.oldLayout        = VK_IMAGE_LAYOUT_UNDEFINED;
barrier.newLayout        = VK_IMAGE_LAYOUT_COLOR_ATTACHMENT_OPTIMAL;
barrier.image            = swapchainImage;
barrier.subresourceRange = { VK_IMAGE_ASPECT_COLOR_BIT, 0, 1, 0, 1 };
```

**After rendering** — `COLOR_ATTACHMENT_OPTIMAL -> PRESENT_SRC_KHR`:
```cpp
VkImageMemoryBarrier2 barrier{};
barrier.sType            = VK_STRUCTURE_TYPE_IMAGE_MEMORY_BARRIER_2;
barrier.srcStageMask     = VK_PIPELINE_STAGE_2_COLOR_ATTACHMENT_OUTPUT_BIT;
barrier.srcAccessMask    = VK_ACCESS_2_COLOR_ATTACHMENT_WRITE_BIT;
barrier.dstStageMask     = VK_PIPELINE_STAGE_2_NONE;
barrier.dstAccessMask    = VK_ACCESS_2_NONE;
barrier.oldLayout        = VK_IMAGE_LAYOUT_COLOR_ATTACHMENT_OPTIMAL;
barrier.newLayout        = VK_IMAGE_LAYOUT_PRESENT_SRC_KHR;
barrier.image            = swapchainImage;
barrier.subresourceRange = { VK_IMAGE_ASPECT_COLOR_BIT, 0, 1, 0, 1 };
```

Both barriers: set `srcQueueFamilyIndex = VK_QUEUE_FAMILY_IGNORED`, `dstQueueFamilyIndex = VK_QUEUE_FAMILY_IGNORED`.

### Submission via `vkQueueSubmit2` (sync2)

Use the Vulkan 1.3 `vkQueueSubmit2` (not the legacy `vkQueueSubmit`). Key structs:

```cpp
VkCommandBufferSubmitInfo cmdSubmitInfo{};
cmdSubmitInfo.sType         = VK_STRUCTURE_TYPE_COMMAND_BUFFER_SUBMIT_INFO;
cmdSubmitInfo.commandBuffer = cmd;

VkSemaphoreSubmitInfo waitInfo{};
waitInfo.sType     = VK_STRUCTURE_TYPE_SEMAPHORE_SUBMIT_INFO;
waitInfo.semaphore = imageAvailableSemaphore;
waitInfo.stageMask = VK_PIPELINE_STAGE_2_COLOR_ATTACHMENT_OUTPUT_BIT;

VkSemaphoreSubmitInfo signalInfo{};
signalInfo.sType     = VK_STRUCTURE_TYPE_SEMAPHORE_SUBMIT_INFO;
signalInfo.semaphore = renderFinishedSemaphore;
signalInfo.stageMask = VK_PIPELINE_STAGE_2_ALL_GRAPHICS_BIT;

VkSubmitInfo2 submitInfo{};
submitInfo.sType                    = VK_STRUCTURE_TYPE_SUBMIT_INFO_2;
submitInfo.waitSemaphoreInfoCount   = 1;
submitInfo.pWaitSemaphoreInfos      = &waitInfo;
submitInfo.commandBufferInfoCount   = 1;
submitInfo.pCommandBufferInfos      = &cmdSubmitInfo;
submitInfo.signalSemaphoreInfoCount = 1;
submitInfo.pSignalSemaphoreInfos    = &signalInfo;

vkQueueSubmit2(graphicsQueue, 1, &submitInfo, renderFence);
```

### Pipeline Creation for Dynamic Rendering

Critical: chain `VkPipelineRenderingCreateInfo` in `VkGraphicsPipelineCreateInfo.pNext`:

```cpp
VkPipelineRenderingCreateInfo renderingCreateInfo{};
renderingCreateInfo.sType                   = VK_STRUCTURE_TYPE_PIPELINE_RENDERING_CREATE_INFO;
renderingCreateInfo.colorAttachmentCount    = 1;
renderingCreateInfo.pColorAttachmentFormats = &swapchainFormat; // VK_FORMAT_B8G8R8A8_SRGB

VkGraphicsPipelineCreateInfo pipelineInfo{};
pipelineInfo.pNext      = &renderingCreateInfo;
pipelineInfo.renderPass = VK_NULL_HANDLE;  // NO render pass
// ... rest of pipeline state ...
```

### GLSL Shaders — Hardcoded Triangle

**`assets/shaders/triangle.vert`**:
```glsl
#version 450

// Vulkan NDC: X right, Y down, Z [0,1]
const vec2 positions[3] = vec2[3](
    vec2( 0.0, -0.5),   // top center
    vec2( 0.5,  0.5),   // bottom right
    vec2(-0.5,  0.5)    // bottom left
);

const vec3 colors[3] = vec3[3](
    vec3(1.0, 0.0, 0.0),  // red
    vec3(0.0, 1.0, 0.0),  // green
    vec3(0.0, 0.0, 1.0)   // blue
);

layout(location = 0) out vec3 fragColor;

void main()
{
    gl_Position = vec4(positions[gl_VertexIndex], 0.0, 1.0);
    fragColor   = colors[gl_VertexIndex];
}
```

**`assets/shaders/triangle.frag`**:
```glsl
#version 450

layout(location = 0) in  vec3 fragColor;
layout(location = 0) out vec4 outColor;

void main()
{
    outColor = vec4(fragColor, 1.0);
}
```

### Shader Compilation

Create `tools/shader_compile.sh`:
```bash
#!/usr/bin/env bash
# Compile all GLSL shaders to SPIR-V
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
SHADER_DIR="$SCRIPT_DIR/../assets/shaders"

for shader in "$SHADER_DIR"/*.vert "$SHADER_DIR"/*.frag "$SHADER_DIR"/*.comp; do
    [ -f "$shader" ] || continue
    echo "Compiling $(basename "$shader")"
    glslangValidator -V --target-env vulkan1.3 "$shader" -o "${shader}.spv"
done
```

Also add CMake custom commands so shaders compile during the build. Use `find_program(GLSLANG glslangValidator)` or `find_program(GLSLC glslc)`.

### SPIR-V File Path Resolution

The dev agent must decide how `Renderer` finds `.spv` files at runtime. Options:
1. **Preferred**: CMake copies `.spv` files to the build output directory next to the executable. Use `configure_file()` or `add_custom_command(POST_BUILD COMMAND ${CMAKE_COMMAND} -E copy ...)`.
2. Hardcode a relative path from executable to `assets/shaders/` (fragile).
3. CMake-generated config header with `SHADER_DIR` path (reliable but adds complexity).

Option 1 is recommended for simplicity.

### Swapchain Recreation

`VulkanContext` currently has no `recreateSwapchain()` method. This story must add one. Pattern:

```cpp
Result<void> VulkanContext::recreateSwapchain(game::Window& window)
{
    vkDeviceWaitIdle(m_device);

    // Destroy old image views
    for (auto view : m_swapchainImageViews)
        vkDestroyImageView(m_device, view, nullptr);

    // Rebuild via vk-bootstrap (pass old swapchain for reuse)
    vkb::SwapchainBuilder builder{m_physicalDevice, m_device, m_surface};
    int fbWidth, fbHeight;
    window.getFramebufferSize(fbWidth, fbHeight);

    auto result = builder
        .set_old_swapchain(m_swapchain)
        .set_desired_format({VK_FORMAT_B8G8R8A8_SRGB, VK_COLOR_SPACE_SRGB_NONLINEAR_KHR})
        .set_desired_present_mode(VK_PRESENT_MODE_FIFO_KHR)
        .set_desired_extent(fbWidth, fbHeight)
        .build();

    // Destroy old swapchain AFTER building new one (old was passed to builder)
    vkDestroySwapchainKHR(m_device, m_swapchain, nullptr);

    // Update members from new swapchain...
}
```

**Option A (Recommended)**: Create a `GameApp` subclass of `GameLoop` in `game/src/`:

class GameApp : public voxel::game::GameLoop

### Anti-Patterns to Avoid

- **DO NOT** create `VkRenderPass` or `VkFramebuffer` — use dynamic rendering exclusively
- **DO NOT** use `vkCmdBeginRenderingKHR` — use core `vkCmdBeginRendering` (Vulkan 1.3)
- **DO NOT** use `vkCmdPipelineBarrier2KHR` — use core `vkCmdPipelineBarrier2`
- **DO NOT** use `vkQueueSubmit` — use `vkQueueSubmit2` (sync2)
- **DO NOT** include `<vulkan/vulkan.h>` — use `<volk.h>` exclusively
- **DO NOT** link to `Vulkan::Vulkan` CMake target — volk replaces the loader
- **DO NOT** create fences without `VK_FENCE_CREATE_SIGNALED_BIT` — first frame would deadlock
- **DO NOT** skip image layout transitions — dynamic rendering does NOT transition layouts
- **DO NOT** store `VkShaderModule` after pipeline creation — destroy them immediately
- **DO NOT** use `std::chrono` for timing — GLFW timing already established in Story 2.1
- **DO NOT** put game-layer code (GameApp subclass) in the engine library — it goes in `game/src/`

- Present mode: `VK_PRESENT_MODE_FIFO_KHR` (vsync, universally available)

- `Window::create()` and `VulkanContext::create()` both use factory+Result pattern
